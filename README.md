Atividade prática com o objetivo de simular um sistema de autenticação seguro utilizado pela empresa fictícia Cyber-SENAI. A proposta consiste em criar um script capaz de controlar o acesso de usuários por meio de login e senha, permitindo no máximo três tentativas antes de bloquear completamente o sistema. O principal foco da atividade é a utilização da estrutura de repetição do...while, garantindo que o usuário realize pelo menos uma tentativa de login e que novas tentativas só ocorram em caso de erro, desde que ainda existam chances disponíveis e que a operação não tenha sido cancelada. O sistema deve conter uma base de dados composta por um array de objetos com dez usuários previamente cadastrados, cada um com seu respectivo login e senha. Além disso, é necessário implementar uma variável de controle que inicia com o valor três e é decrementada a cada tentativa inválida. Durante a execução, o sistema deve validar se as credenciais inseridas correspondem a algum usuário da base de dados. Caso o login seja bem-sucedido, a mensagem exibida deve ser “Acesso Liberado”. Em caso de erro, o sistema deve informar ao usuário quantas tentativas ainda restam. Se todas as tentativas forem utilizadas sem sucesso, o sistema deve exibir a mensagem “SISTEMA BLOQUEADO - Procure o Suporte”. Esse projeto permite o desenvolvimento de habilidades importantes como lógica de programação, controle de fluxo, uso de estruturas de repetição, manipulação de arrays e validação de dados, além de simular um cenário real de segurança digital.
